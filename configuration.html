Configuration

Now that your plugin is installed, here is how to configure it.

Credentials

The first information you'll need to provide are your Algolia credentials to create the link between Algolia and your store.

They're available in the Credentials tab of your Algolia dashboard.

From that screen you will be able to copy/paste the following necessary fields from the Algolia dashboard to your application:

  • Application ID
  • Search-Only API Key
  • Admin API Key

You can use the little buttons on the right of the fields to copy them to your clipboard directly.

On error, you'll be prompted with an invalid credentials message. In that case, double-check that you copied the correct fields, you might have copied the Monitoring API key instead of the Admin one.

On success, you'll be prompted with a success message and you'll now have access to the other configuration tabs.

Indexing

As soon as your credentials have been updated, we'll start indexing your content.
An index is a way of representing your data, in Algolia, that has been optimized to answer to search queries almost instantly. You can see those indices in the Indices tab of your Algolia dashboard.

We currently support indexing of 2 types of data from Shopify:

  • Products (variants)
  • Collections

In the Indexing tab, you'll be able to manage which data you want searchable.
Simply toggle the Enable buttons and save to start or stop indexing a type of data.

To get more info on what data is actually pushed to Algolia, check out the Schemas section of the documentation.
This integration also supports metafields out of the box. You'll need to specify which one you want to add to your indices in the Metafields sections of this page. You'll get more information about this in the Metafields section of the documentation.

Display

Now that the you've set the indexing up, the next step is to install our front-end logic in your theme and configure it. This will be done in the Display tab of the application.

On the top right corner, you can to chose which theme you want to use.

Installing Algolia in your theme

Duplicating your theme

If you're on a live store, we recommend you to use a duplicate of this theme. This will allow you to preview the changes, check if everything is working fine and tweak the CSS if needed before putting it live.

To create one, head to Online store > Themes > ... in the top right of the theme entry > Duplicate.

This will take a bit of time, just wait for the loading wheel to stop spinning.

As soon as it is completed, you can go back to your app, reload the page and chose your new theme.

Once you're happy with every change you've done, you can simply head back to the Themes tab of your Store and click Publish theme.

Installation process

Depending on the theme you're installing Algolia in and on whether Algolia is already installed or not, you might get a few confirmation steps. Please read them carefully.

You'll need to help us on the layouts step. We're installing some assets in your theme, but in order to use them, we'll need to add a bit of code inside the <head> tags of your layout files. Simply select all the layouts in which you want Algolia to be installed.

A blue arrow means that we'll automatically add our snippet of code inside a layout.

A yellow double arrow indicates that we'll replace an already existing Algolia snippet.

In case of error, you'll see a red cross next to the layout name and won't be able to continue to the next step. This indicated a scenario where we haven't been able to automatically insert our code snippet.
The solution in that case will be easy : we'll provide you with a small snippet of code and links to open the editor for each of the failing files. Just copy this snippet and paste it inside the <head> tags of those layout files.
Then, uncheck them to see the error disappear and be able to continue the installation.

Front-end configuration

Once you've installed Algolia in your theme, you'll have access to a few configuration options to change the default behaviour.

You'll also be able to chose if you want to use the autocompletion menu and the search results page (disabled by default — more info in Search page).

If those configuration options aren't enough to fit your needs, you can still dive in our front-end code, as desribed in the Front-end Customization part of this documentation.

Sorting

This tab is only useful if you've decided to replace the search page of your store with Algolia.

There are plenty of attributes on which you might want your users to be able to sort on.
In the Sorting tab, you can chose which sort orders you want to activate, how you want them to be named and which ones to display first.

You can also sort on a metafield. Just check the product metafield as a sort order in the Indexing tab, save and it will be available in the Sorting tab.
In order to deliver our speed, our ranking is done at indexing time. Which means that for each sort order, we create a new index which contains the same amount of records as the main one.
You should double-check that an increase in amount of records wouldn't exceed your plan quota.

Sorting is obviously a really important topic for us. We have a full guide about how our relevance works here.

Faceting

This tab is only useful if you've decided to replace the search page of your store with Algolia.

Faceting is the technical word to describe navigation using filters. This is mostly useful for refining a search, but can also a be a way to browse using search.

If you want more information about faceting, you should check the full guide in Algolia's documentation.

For instance, with this integration, you can filter by type, brand or price.

For each facet, you can chose its position in the list, its displayed name and what type of facet it is.
We currently have 4 available types:

  • slider: A simple slider for numeric values
  • conjunctive: Filters using AND between its refinements
  • disjunctive: Filters using OR between its refinements
  • menu: Allows for only one selected refinement but lists other possible values
You can also use a metafield as a facet. Just check the product metafield as a facet in the Indexing tab, save and it will be available in the Faceting tab.
